# Internationalization (i18n)

## Setup

The project uses `i18next` + `react-i18next` with three languages:

| Code | Language |
|------|----------|
| `uk` | Ukrainian (fallback) |
| `en` | English |
| `pl` | Polish |

Configuration is in `src/i18n/config.ts`.

## Language Detection

Detection order (via `i18next-browser-languagedetector`):

1. `localStorage` key: `way_store_lang`
2. Browser `navigator.language`

Fallback: `uk` (Ukrainian).

## Namespaces

Translations are split into namespaces matching page/feature areas:

| Namespace | Scope |
|-----------|-------|
| `common` | Shared UI strings (default namespace) |
| `home` | Home page |
| `store` | Store/catalog |
| `goods` | Product detail page |
| `cart` | Shopping cart |
| `auth` | Login/register modals |
| `account` | User account pages |
| `checkout` | Checkout flow |
| `enums` | Enum labels (metals, stones, etc.) |
| `about` | About page |
| `collections` | Collections page |
| `faq` | FAQ sections |
| `offer` | Public offer page |
| `privacy` | Privacy policy page |

## File Location

```
src/i18n/locales/
├── uk/    # Ukrainian translations
│   ├── common.json
│   ├── home.json
│   └── ...
├── en/    # English translations
└── pl/    # Polish translations
```

## Usage

```tsx
import { useTranslation } from 'react-i18next'

const { t } = useTranslation('store')

<h1>{t('title')}</h1>
<p>{t('filters.metal')}</p>
```

## Adding a New Translation Key

1. Add the key to `src/i18n/locales/uk/<namespace>.json`
2. Add the same key to `en/` and `pl/` locale files
3. Use `t('key')` with the appropriate namespace

## Adding a New Language

1. Create a new folder under `src/i18n/locales/<code>/`
2. Copy all JSON files from `uk/` and translate them
3. Import all files in `src/i18n/config.ts`
4. Add the language to the `resources` object
